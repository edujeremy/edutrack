## 1. 사전 준비

이 단계에서는 필요한 프로그램들을 설치합니다. 마우스로 클릭하기만 하면 됩니다.

### 1-1. Node.js 설치 (필수)

**Node.js는 웹 애플리케이션을 돌려주는 프로그램입니다.**

**단계:**

1. 브라우저를 열고 https://nodejs.org 로 접속합니다.
2. 화면에 보이는 두 개의 큰 버튼 중 왼쪽 버튼 **"LTS" 버튼**을 클릭합니다.
   - LTS는 안정적인 버전을 의미합니다.
   - 버전 번호는 상관없습니다 (예: 20.x, 22.x 등).
3. 다운로드가 시작됩니다. 기다립니다.
4. 다운로드 폴더에서 파일을 더블클릭해서 실행합니다.
5. 설치 마법사가 뜹니다. 계속 **"Next"** 버튼을 클릭합니다.
6. 마지막에 **"Install"** 버튼을 클릭합니다.
7. 완료 후 **"Finish"** 버튼을 클릭합니다.

**설치 확인 방법:**
- 윈도우: 시작 메뉴에서 "명령 프롬프트"를 검색해서 엽니다.
- 맥: 응용 프로그램 → 유틸리티 → 터미널을 엽니다.
- 다음 명령을 복사해서 붙여넣고 Enter를 누릅니다:
  ```
  node --version
  ```
- 버전 번호가 나타나면 성공입니다. (예: v20.11.0)

---

### 1-2. Git 설치 (필수)

**Git은 프로젝트 코드를 저장하고 관리하는 프로그램입니다.**

**단계:**

1. 브라우저에서 https://git-scm.com 로 접속합니다.
2. **"Download for Windows"** 또는 **"Download for Mac"** 버튼을 클릭합니다.
   (자동으로 당신의 운영체제를 인식합니다)
3. 파일을 다운로드합니다.
4. 다운로드 폴더에서 파일을 더블클릭해서 실행합니다.
5. 설치 마법사가 나타납니다. 계속 **"Next"** 버튼을 클릭합니다.
6. "Choose the default editor used by Git" 화면이 나타나면, 선택된 것을 그대로 두고 **"Next"**를 클릭합니다.
7. 계속 **"Next"**를 클릭하고 마지막에 **"Install"**을 클릭합니다.
8. **"Finish"**를 클릭해서 완료합니다.

**설치 확인 방법:**
- 명령 프롬프트(윈도우) 또는 터미널(맥)을 엽니다.
- 다음을 복사해서 붙여넣고 Enter를 누릅니다:
  ```
  git --version
  ```
- 버전 번호가 나타나면 성공입니다. (예: git version 2.42.0)

---

### 1-3. VS Code 설치 (선택, 하지만 추천)

**VS Code는 코드를 편집할 때 유용한 프로그램입니다. 이 가이드에서는 필수는 아니지만 설치하면 작업이 더 쉬워집니다.**

**단계:**

1. 브라우저에서 https://code.visualstudio.com 로 접속합니다.
2. **"Download"** 버튼을 클릭합니다.
3. 당신의 운영체제에 맞는 버전을 선택합니다.
4. 파일을 다운로드합니다.
5. 다운로드 폴더에서 파일을 더블클릭해서 실행합니다.
6. 설치 마법사가 나타나면 계속 **"Next"**를 클릭합니다.
7. **"Install"**을 클릭하고 완료합니다.

---

### 1-4. GitHub 계정 만들기 (필수)

**GitHub는 인터넷에 프로젝트를 저장하는 서비스입니다.**

**단계:**

1. 브라우저에서 https://github.com 로 접속합니다.
2. 화면 오른쪽 위의 **"Sign up"** 버튼을 클릭합니다.
3. 이메일 주소를 입력합니다.
4. **"Create account"** 버튼을 클릭합니다.
5. 비밀번호를 입력합니다.
6. 사용자명(username)을 입력합니다. (예: jeremy123)
   - 이 이름은 나중에 필요하니 기억해두세요.
7. 이메일 업데이트 선택: **"No"**를 선택합니다.
8. **"Create account"**를 클릭합니다.
9. 이메일 검증 코드가 메일로 옵니다. 메일을 확인하고 코드를 입력합니다.
10. 가입이 완료됩니다.

---

### 1-5. Vercel 계정 만들기 (필수)

**Vercel은 만든 웹 사이트를 인터넷에 올리는 서비스입니다.**

**단계:**

1. 브라우저에서 https://vercel.com 로 접속합니다.
2. **"Sign Up"** 버튼을 클릭합니다.
3. **"Continue with GitHub"** 버튼을 클릭합니다.
4. GitHub 로그인이 필요하면 로그인합니다.
5. Vercel에 GitHub 접근 권한을 주는지 묻습니다. **"Authorize Vercel"** 버튼을 클릭합니다.
6. 가입이 완료됩니다.

## 2. Supabase 프로젝트 설정

**Supabase는 데이터베이스를 관리하는 서비스입니다. 학생, 반, 수강료 정보 등이 저장됩니다.**

### 2-1. Supabase 가입 및 로그인

**단계:**

1. 브라우저에서 https://supabase.com 로 접속합니다.
2. 화면 오른쪽 위의 **"Sign In"** 버튼을 클릭합니다.
3. **"Continue with GitHub"** 버튼을 클릭합니다.
4. GitHub 로그인을 진행합니다.
5. Supabase에 GitHub 접근 권한을 주는지 묻습니다. **"Authorize"** 버튼을 클릭합니다.

---

### 2-2. 새 프로젝트 만들기

**단계:**

1. 로그인 후, 화면에 **"New Project"** 또는 **"Create a new project"** 버튼이 보입니다. 클릭합니다.
2. 프로젝트 이름을 입력합니다. **정확히 다음을 입력하세요:**
   ```
   edutrack
   ```
3. 비밀번호를 입력합니다. (당신이 정하는 것입니다)
   - 예: 학원1234!
   - 이 비밀번호는 **기억해두세요**. 나중에 필요합니다.
4. Region(지역)을 선택합니다. **"Northeast Asia (Tokyo)"** 를 찾아서 클릭합니다.
5. **"Create new project"** 버튼을 클릭합니다.
6. 프로젝트가 생성되기를 기다립니다. (2-3분 소요)

---

### 2-3. 데이터베이스 테이블 만들기

**프로젝트 생성이 완료되면, 화면 왼쪽에 메뉴가 보입니다.**

**단계:**

1. 왼쪽 메뉴에서 **"SQL Editor"**를 클릭합니다.
2. 오른쪽 위의 **"New Query"** 버튼을 클릭합니다.
3. 큰 텍스트 상자가 나타납니다. 이 상자를 클릭합니다.
4. 다음 파일의 **전체 내용**을 복사합니다:
   ```
   supabase-schema.sql
   ```
   (이 파일은 edutrack 프로젝트 폴더에 있습니다)
5. 복사한 내용을 텍스트 상자에 **붙여넣습니다**.
6. 오른쪽 위의 **"Run"** 버튼을 클릭합니다.
7. 완료 메시지가 나타나면 성공입니다.

**그림으로 설명:**
```
┌─────────────────────────────────────┐
│ SQL Editor 화면                      │
│                                     │
│ ┌──────────────────────────────┐   │
│ │ [New Query] 버튼              │   │
│ └──────────────────────────────┘   │
│                                     │
│ ┌──────────────────────────────┐   │
│ │                              │   │
│ │  SQL 코드가 들어가는 곳 👈    │   │
│ │  (여기에 붙여넣기)           │   │
│ │                              │   │
│ └──────────────────────────────┘   │
│                                     │
│ [Run] 버튼 👈 클릭하기             │
└─────────────────────────────────────┘
```

### 2-4. API 키와 URL 복사하기 (매우 중요!)

**이 정보들은 나중에 필요하니 메모장에 저장해두세요.**

**단계:**

1. Supabase 대시보드로 돌아갑니다.
2. 왼쪽 메뉴에서 **"Settings"**를 클릭합니다.
3. **"API"**를 클릭합니다.
4. 다음 정보들을 각각 복사해서 메모장에 저장합니다:

   **A. Project URL 복사:**
   - "Project URL" 옆의 텍스트를 클릭합니다. (예: https://xxxxx.supabase.co)
   - 자동으로 복사됩니다. Ctrl+C (또는 Cmd+C)를 누르거나 마우스 우클릭해서 복사합니다.
   - 메모장에 붙여넣습니다.

   **B. anon public key 복사:**
   - "Project API keys" 섹션에서 "anon" 옆의 숨겨진 키를 찾습니다.
   - 키 옆의 **복사 아이콘(📋)**을 클릭합니다.
   - 메모장에 붙여넣습니다. 라벨을 붙여두세요: "anon key"

   **C. service_role key 복사:** (비밀번호 같은 것입니다)
   - 같은 페이지에서 "service_role" 키를 찾습니다.
   - 키 옆의 **복사 아이콘(📋)**을 클릭합니다.
   - 메모장에 붙여넣습니다. 라벨을 붙여두세요: "service_role key"

**메모장 템플릿:**
```
=== Supabase 정보 ===
Project URL: https://xxxxx.supabase.co
anon key: eyJxxxxx...
service_role key: eyJxxxxx...
```

## 3. Resend (이메일) 설정

**Resend는 학부모에게 이메일(연락, 공지 등)을 보내는 서비스입니다.**

### 3-1. Resend 가입

**단계:**

1. 브라우저에서 https://resend.com 로 접속합니다.
2. **"Sign Up"** 버튼을 클릭합니다.
3. 이메일 주소를 입력합니다.
4. 비밀번호를 입력합니다.
5. **"Create account"** 버튼을 클릭합니다.
6. 이메일 검증을 합니다. (메일을 확인하고 링크를 클릭)

---

### 3-2. API 키 만들고 복사하기

**단계:**

1. Resend 대시보드에서 왼쪽 메뉴를 봅니다.
2. **"API Keys"**를 클릭합니다.
3. 오른쪽 위의 **"Create API Key"** 또는 **"New API Key"** 버튼을 클릭합니다.
4. 이름을 입력합니다. (예: edutrack)
5. **"Create"** 버튼을 클릭합니다.
6. 새로운 API 키가 생성됩니다. **복사 아이콘(📋)**을 클릭해서 복사합니다.
7. 메모장에 붙여넣습니다. 라벨을 붙여두세요: "Resend API Key"

**메모장 추가:**
```
=== Resend 정보 ===
Resend API Key: re_xxxxx...
```

## 4. 프로젝트 로컬 설정

**이제 당신의 컴퓨터에 프로젝트를 설정합니다.**

### 4-1. 프로젝트 폴더 준비

**단계:**

1. 프로젝트 폴더(edutrack)를 받습니다.
   - 이 폴더는 누군가가 압축 파일로 주었을 것입니다.
   - 압축을 풀어서 바탕화면에 놓습니다.

2. 또는 GitHub에서 받는 경우:
   - 프로젝트를 GitHub에 올린 사람의 링크를 클릭합니다.
   - 초록색 **"Code"** 버튼을 클릭합니다.
   - **"Download ZIP"**을 클릭합니다.
   - 다운로드된 파일을 압축 해제합니다.
   - 바탕화면에 놓습니다.

**결과:** 바탕화면에 "edutrack" 폴더가 있어야 합니다.

---

### 4-2. 환경 설정 파일 만들기

**.env.local 파일**은 민감한 정보를 저장하는 파일입니다.

**단계:**

1. VS Code를 엽니다. (또는 메모장)
2. 메뉴에서 **"File"** → **"Open Folder"**를 클릭합니다.
3. 바탕화면의 **edutrack 폴더**를 선택해서 엽니다.
4. 왼쪽에 폴더 구조가 보입니다.
5. **.env.local.example** 파일을 찾습니다.
6. 파일을 마우스 우클릭합니다.
7. **"Copy"**를 클릭합니다.
8. 빈 공간에 우클릭해서 **"Paste"**를 클릭합니다.
9. 새로 생긴 파일의 이름을 **.env.local**로 바꿉니다.
   - 우클릭 → **"Rename"** → **.env.local** 입력

---

### 4-3. 환경 설정 파일에 정보 입력

**.env.local 파일**을 메모장이나 VS Code로 엽니다.

**단계:**

1. 파일을 더블클릭해서 엽니다. (VS Code가 실행됩니다)
2. 파일 내용을 봅니다. 다음과 같은 형태입니다:
   ```
   NEXT_PUBLIC_SUPABASE_URL=
   NEXT_PUBLIC_SUPABASE_ANON_KEY=
   SUPABASE_SERVICE_ROLE_KEY=
   RESEND_API_KEY=
   ```

3. 각 줄의 **=** 뒤에 정보를 붙여넣습니다:

   **예시:**
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJxxxxx...
   SUPABASE_SERVICE_ROLE_KEY=eyJxxxxx...
   RESEND_API_KEY=re_xxxxx...
   ```

4. **Ctrl+S** (또는 **Cmd+S**)를 눌러서 저장합니다.

**주의:**
- 값 앞뒤로 따옴표(")를 붙이지 마세요.
- 공백이 없어야 합니다.
- 정보가 제대로 붙여졌는지 확인하세요.

---

### 4-4. 프로그램 설치 (npm install)

**이제 필요한 라이브러리들을 설치합니다.**

**단계:**

1. 명령 프롬프트(윈도우) 또는 터미널(맥)을 엽니다.
2. 다음 명령을 입력해서 바탕화면으로 이동합니다:
   ```
   cd Desktop
   ```
   엔터를 누릅니다.

3. 다음 명령을 입력합니다:
   ```
   cd edutrack
   ```
   엔터를 누릅니다.

4. 이제 edutrack 폴더 안에 있습니다. 확인해보면 프롬프트에 "edutrack"이 포함되어 있을 것입니다.

5. 다음 명령을 입력합니다:
   ```
   npm install
   ```
   엔터를 누릅니다.

6. 설치가 시작됩니다. **기다립니다.** (2-5분 소요)
   - 많은 파일들이 다운로드됩니다.
   - 중간에 경고 메시지가 나올 수 있습니다. 무시해도 괜찮습니다.
   - "added XXX packages"라는 메시지가 나타나면 완료입니다.

---

### 4-5. 로컬에서 실행해보기

**이제 당신의 컴퓨터에서 웹 사이트를 실행합니다.**

**단계:**

1. 같은 터미널에서 다음 명령을 입력합니다:
   ```
   npm run dev
   ```
   엔터를 누릅니다.

2. 다음과 같은 메시지가 나타납니다:
   ```
   > next dev
   ▲ Next.js 15.x.x
   - Local:        http://localhost:3000
   ```

3. 브라우저를 엽니다.

4. 주소 표시줄에 다음을 입력합니다:
   ```
   http://localhost:3000
   ```
   엔터를 누릅니다.

5. EduTrack 웹 사이트가 로드됩니다!

**확인 사항:**
- 로그인 화면이 보이나요?
- 에러가 나나요? → 아래의 "문제 해결" 섹션을 봅니다.

**브라우저 창을 종료하고 싶을 때:**
- 터미널에서 **Ctrl+C**를 누르면 서버가 중지됩니다.

## 5. GitHub에 올리기

**이제 코드를 GitHub에 저장합니다. 이것을 "푸시(Push)"라고 부릅니다.**

### 5-1. GitHub Repository 만들기

**단계:**

1. GitHub.com에 로그인합니다.
2. 화면 오른쪽 위에 **"+"** 버튼이 있습니다. 클릭합니다.
3. **"New Repository"**를 클릭합니다.
4. "Repository name" 필드에 입력합니다:
   ```
   edutrack
   ```
5. Description은 (선택사항) 입력합니다:
   ```
   EduTrack - 학원 관리 자동화 시스템
   ```
6. **"Private"**를 선택합니다. (보안 목적)
7. 아래의 "Initialize this repository with:" 섹션은 **체크 해제합니다.**
8. **"Create repository"** 버튼을 클릭합니다.
9. 새로운 페이지가 나타나고, 다음과 같은 명령들을 볼 수 있습니다.

---

### 5-2. 터미널에서 Git 명령 실행

**이 단계는 복사-붙여넣기만 하면 됩니다!**

**단계:**

1. 터미널/명령 프롬프트를 엽니다. (edutrack 폴더 안에 있어야 합니다)
   - 프롬프트가 "edutrack"을 포함하고 있는지 확인하세요.

2. 다음 명령들을 **차례대로** 복사해서 붙여넣습니다:

   **첫 번째 명령:**
   ```
   git init
   ```
   엔터를 누릅니다.

   **두 번째 명령:**
   ```
   git add .
   ```
   엔터를 누릅니다.

   **세 번째 명령:**
   ```
   git commit -m "Initial commit"
   ```
   엔터를 누릅니다.

   **네 번째 명령:**
   ```
   git branch -M main
   ```
   엔터를 누릅니다.

3. 이제 GitHub에서 repository를 만들었을 때 나온 명령을 봅니다.
   - 다음과 같은 형태입니다:
     ```
     git remote add origin https://github.com/YOUR_USERNAME/edutrack.git
     ```
   - **YOUR_USERNAME을 당신의 GitHub 사용자명으로 바꿉니다.**
   - 이 명령을 복사해서 터미널에 붙여넣습니다.
   - 엔터를 누릅니다.

4. 마지막 명령:
   ```
   git push -u origin main
   ```
   엔터를 누릅니다.

5. GitHub 로그인이 필요할 수 있습니다:
   - 브라우저 창이 열리면 로그인합니다.
   - 또는 토큰을 입력하라는 창이 나올 수 있습니다. (GitHub Personal Access Token)

6. 완료되면 다음과 같은 메시지가 나타납니다:
   ```
   * [new branch]      main -> main
   ```

**축하합니다! 코드가 GitHub에 올려졌습니다!**

   **세 번째: SUPABASE_SERVICE_ROLE_KEY**
   - Key: `SUPABASE_SERVICE_ROLE_KEY`
   - Value: (메모장에서 복사한 service_role key)
   - **"Add"** 버튼을 클릭합니다.

   **네 번째: RESEND_API_KEY**
   - Key: `RESEND_API_KEY`
   - Value: (메모장에서 복사한 Resend API Key)

## 7. Supabase Auth 설정

**이제 로그인 기능이 정상 작동하도록 Supabase를 설정합니다.**

### 7-1. Site URL 변경

**단계:**

1. Supabase 대시보드에 접속합니다.
2. "edutrack" 프로젝트를 클릭해서 엽니다.
3. 왼쪽 메뉴에서 **"Authentication"**을 클릭합니다.
4. **"Providers"**를 클릭합니다.
5. 화면 상단의 **"URL Configuration"**을 찾아서 클릭합니다.
6. **"Site URL"** 필드를 찾습니다.
7. 기본값(예: http://localhost:3000)을 지웁니다.
8. Vercel에서 받은 배포 URL을 입력합니다. (예: https://edutrack-xxxxx.vercel.app)
9. 오른쪽 아래의 **"Save"** 버튼을 클릭합니다.

---

### 7-2. Redirect URLs 추가

**단계:**

1. 같은 "URL Configuration" 페이지에 있습니다.
2. **"Redirect URLs"** 섹션을 찾습니다.
3. **"Add URI"** 버튼을 클릭합니다.
4. 다음을 입력합니다:
   ```
   https://edutrack-xxxxx.vercel.app/auth/callback
   ```
   (xxxxx는 당신의 배포 URL에 맞게 바꿉니다)

5. **"Save"** 버튼을 클릭합니다.

## 9. 문제 해결 (FAQ)

**배포 중에 문제가 발생했나요? 여기서 해결책을 찾아보세요.**

### 문제 1: npm install 에러

**증상:**
```
error: node_modules or package lock not found
또는
Could not find module '@supabase/supabase-js'
```

**해결 방법:**

1. Node.js 버전 확인:
   ```
   node --version
   ```
   버전이 14.x 이상인지 확인합니다.

2. npm 캐시 삭제:
   ```
   npm cache clean --force
   ```

3. node_modules 폴더 삭제:
   - edutrack 폴더에서 "node_modules"라는 폴더를 찾습니다.
   - 삭제합니다.

4. 다시 설치:
   ```
   npm install
   ```

---

### 문제 2: 로그인이 안 됨

**증상:**
- 회원가입 후 로그인이 안 됨
- "Authentication failed" 에러
- Supabase URL/Key 관련 에러

**해결 방법:**

1. **.env.local 파일 확인:**
   - VS Code에서 .env.local을 엽니다.
   - Supabase URL이 정확히 입력되었는지 확인합니다.
   - anon key가 정확히 입력되었는지 확인합니다.
   - 공백이나 따옴표가 없는지 확인합니다.

2. **파일 저장 후 다시 실행:**
   ```
   npm run dev
   ```

3. **Supabase Auth 설정 확인:**
   - Supabase → Authentication → URL Configuration 에서:
   - Site URL이 배포 URL인지 확인합니다.
   - Redirect URLs이 올바르게 설정되었는지 확인합니다.

---

### 문제 3: 이메일이 안 옴

**증상:**
- 회원가입 확인 메일이 오지 않음
- 비밀번호 초기화 메일이 오지 않음

**해결 방법:**

1. **Resend API Key 확인:**
   - .env.local에서 RESEND_API_KEY가 정확히 입력되었는지 확인합니다.

2. **Vercel 환경 변수 확인:**
   - Vercel 대시보드 → Settings → Environment Variables
   - RESEND_API_KEY가 올바르게 설정되었는지 확인합니다.

3. **스팸 폴더 확인:**
   - 메일의 스팸/프로모션 폴더를 확인합니다.

4. **Resend 계정 확인:**
   - Resend.com에 로그인합니다.
   - API Key가 활성화되어 있는지 확인합니다.

---

### 문제 4: 빌드 에러 (Vercel에서)

**증상:**
- Vercel 배포 중 "Build failed" 에러
- "Cannot find module" 에러

**해결 방법:**

1. **환경 변수 다시 확인:**
   - Vercel → Settings → Environment Variables
   - 모든 필수 변수가 입력되었는지 확인합니다:
     - NEXT_PUBLIC_SUPABASE_URL
     - NEXT_PUBLIC_SUPABASE_ANON_KEY
     - SUPABASE_SERVICE_ROLE_KEY
     - RESEND_API_KEY

2. **Vercel에서 다시 배포:**
   - Vercel 대시보드에서 "Deployments" 탭을 클릭합니다.
   - 최신 배포 옆의 **"Redeploy"** 버튼을 클릭합니다.

3. **GitHub에 코드 다시 푸시:**
   - 터미널에서:
     ```
     git add .
     git commit -m "Fix environment variables"
     git push origin main
     ```

### 문제 5: 데이터베이스가 비어있음

**증상:**
- 웹 사이트에 데이터가 안 보임
- 학생/반 정보가 없음

**해결 방법:**

   - 실행한 쿼리 목록을 확인합니다.

2. **테이블이 있는지 확인:**
   - Supabase → Table Editor
   - "classes", "students", "payments" 등의 테이블이 있는지 확인합니다.

3. **다시 SQL 실행:**
   - supabase-schema.sql 파일의 내용을 다시 복사합니다.
   - SQL Editor에서 새 쿼리를 만들고 붙여넣습니다.
   - **"Run"** 버튼을 클릭합니다.

### 문제 6: Git 관련 에러

**증상:**
```
fatal: not a git repository
또는
permission denied
```

**해결 방법:**

1. **폴더 위치 확인:**
   - 터미널 프롬프트에 "edutrack"이 포함되어 있는지 확인합니다.

2. **git init 다시 실행:**
   ```
   git init
   ```

3. **GitHub 연결 확인:**

### 문제 7: "localhost:3000이 응답하지 않음" 에러

**증상:**
- `npm run dev` 실행 후 브라우저에서 localhost:3000에 접속할 수 없음
- 에러: "This site can't be reached"

**해결 방법:**

1. **터미널에서 에러 메시지 확인:**
   - 빨간색 에러 메시지가 있는지 봅니다.

2. **포트 확인:**
   - 터미널에 "Local: http://localhost:3000" 메시지가 보이는지 확인합니다.

3. **다시 실행:**
   ```
   npm run dev
   ```

4. **포트 변경 (필요한 경우):**
   ```
   npm run dev -- -p 3001
   ```
   그 다음 http://localhost:3001 로 접속합니다.

### 문제 8: "Cannot connect to Supabase" 에러

**증상:**
- 웹 사이트가 로드되지만 Supabase 연결 실패 에러
- 콘솔에 "Supabase connection error" 메시지

**해결 방법:**

1. **.env.local 다시 확인:**
   - NEXT_PUBLIC_SUPABASE_URL이 정확한지 확인합니다.
   - 끝에 슬래시(/)가 없는지 확인합니다.

2. **Supabase 프로젝트 상태 확인:**
   - Supabase.com에 로그인합니다.
   - 프로젝트가 활성화되어 있는지 확인합니다.
   - "Project Status"가 "Active"인지 확인합니다.

3. **API가 활성화되었는지 확인:**
   - Supabase → Settings → API
   - URL과 key가 제대로 복사되었는지 다시 확인합니다.
